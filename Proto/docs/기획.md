# 🛠️ ProtoEngine Architecture Specification

## 1. Core Engine & Lifecycle (Directory: src/Core)
### 1.1. Application Class
- **Responsibility**: Singleton-based engine controller. Manages initialization, main loop, and shutdown.
- **Lifecycle Methods**:
  - `Init()`: Initialize GLFW, GLAD, and Dear ImGui context.
  - `Run()`: Execute the main game loop (Process Events -> Update -> Render).
  - `Shutdown()`: Clean up resources and terminate libraries.
- **Facade API**: Wrap complex internal logic into simple interfaces like `Engine::Init()`, `Engine::Run()`.

### 1.2. Input System (Directory: src/Core/Input)
- **Responsibility**: Singleton manager for hardware input (Keyboard/Mouse).
- **Mechanism**: Captures GLFW callbacks and stores state bits.
- **Polling API**: `Input::IsKeyPressed(KeyCode)`, `Input::GetMousePosition()`.

---

## 2. Scene & Component System (Directory: src/Scene)
### 2.1. GameObject-Component Pattern
- **GameObject**: A generic container for components.
  - Holds a unique ID, Name, and `std::vector<Component*>`.
  - Supports `AddComponent<T>()`, `GetComponent<T>()` templates.
  - Maintains Parent-Child hierarchy pointers.
- **Component (Base)**: Abstract class for behaviors.
  - Virtual methods: `OnStart()`, `OnUpdate()`, `OnDestroy()`.
  - Reference to owner `GameObject`.

### 2.2. Scene Manager
- **Responsibility**: Manages the lifecycle of all GameObjects in the current scene.
- **Functions**: `CreateGameObject()`, `DestroyGameObject()`.
- **Update Loop**: Iterates through all active GameObjects and calls `Component::OnUpdate()`.

### 2.3. Core Components
- **Transform**: Manages Position, Rotation (Euler/Quaternion), and Scale.
- **MeshRenderer**: Encapsulates VBO/EBO and issues Draw Calls for primitives (Cube, Plane).
- **Camera**: Calculates View and Projection matrices.
- **Collider**: Defines AABB or simple shapes for collision detection.
- **Script**: Base class for user-defined C++ logic.

### 2.4. Space Transformation
- **World Matrix Calculation**: 
  - M_world = M_parent_world * M_local

---

## 3. Rendering Pipeline (Directory: src/Renderer)
### 3.1. Renderer Facade
- **Goal**: Abstract low-level OpenGL calls.
- **Shader Class**: Handles GLSL compilation, linking, and uniform uploading.
- **Vertex Data**: Structured using VBO (Vertex Buffer) and EBO (Element Buffer).

### 3.2. Lighting & Shading
- **Simple Lighting**: Implement Directional Light with Basic Phong Shading.
- **Projection**: Supports Perspective Projection.
  - V_clip = M_projection * M_view * M_model * V_local

---

## 4. Editor Interface (Directory: src/Editor)
### 4.1. Dear ImGui Integration
- **Framework**: Use Dear ImGui (Docking Branch) for the UI layout.
- **Layout**: Flexible docking space for multi-panel management.

### 4.2. Editor Panels
- **Hierarchy Panel**: Visualizes the Scene tree. Supports selection and drag-and-drop parenting.
- **Inspector Panel**: Reflects selected GameObject's components. Allows real-time modification of `Transform` values.
- **Play Mode Control**: Start/Stop button to test game logic. Includes a simple scene state backup/restore system.
