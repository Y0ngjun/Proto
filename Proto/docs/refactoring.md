# 리팩토링 지침서 (Refactoring Guide)

Proto 엔진의 전반적인 코드 품질 향상과 유지보수성 확보를 위한 리팩토링 기준 및 지침입니다.

## 리팩토링 주요 지침 (작업순서 / 사이클)

각 단계를 하나의 **사이클(Cycle)** 로 간주하여 순차적으로 진행합니다.

1. **죽은 코드 및 주석 제거**: 사용하지 않는 함수, 변수, 과거의 주석 처리된 코드를 완전히 삭제합니다.
2. **단일 책임 원칙 (SRP)**: 클래스와 함수가 오직 하나의 책임만 가지도록 분리합니다.
3. **의존성 최소화**: 불필요한 `#include`를 제거하고 전방 선언(Forward Declaration)을 적극 활용합니다.
4. **종합 개선 (아래 세 가지는 한 번에 적용)**
   - 매직 넘버/스트링 상수화
   - 빠른 반환 (Early Return) 적용
   - 스타일 가이드 준수 (네이밍, Allman 스타일 중괄호 등)
5. **md파일 최신화**: 모든 사이클이 완료된 후 변경된 코드에 맞춰 각 md 파일을 최신화합니다. (필요 시 생성/삭제)

---

## 주의사항 (작업 방식)

1. **사이클 단위 진행**: 각 단계 (1~4) 를 나누어 진행합니다. 이때 한 단계를 **사이클**이라고 합니다.
2. **작업 단위 분할**: 한 사이클을 진행할 때 작업 단위를 분할하여 진행합니다. 한 사이클을 총 6단계에 나눠서 진행합니다.
   - `main.cpp`
   - `Asset/`
   - `Core/`
   - `Editor/`
   - `Renderer/`
   - `Scene/`
3. **문서 동기화**: 모든 사이클이 완료된 후 변경된 내용에 맞춰서 각 md 파일을 최신화 합니다.
